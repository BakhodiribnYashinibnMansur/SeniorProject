# Raw String Literals in Go — Senior Level

## 1. Introduction

At the senior level, raw string literals are analyzed through the lens of system design, maintenance, and architectural impact. The questions are: how do raw strings affect codebase organization, how do they perform in large systems, and what architectural patterns make the most of Go's raw string design?

## 3. Raw Strings and Binary Layout

Raw string literals are compile-time constructs. At the machine level:

```go
// This raw string:
const query = `SELECT * FROM users WHERE id = $1`

// Ends up in the binary's .rodata (read-only data) section:
// $ strings -t x ./binary | grep "SELECT"
// 4a2f40 SELECT * FROM users WHERE id = $1

// The variable 'query' on the stack (or in the text segment for constants)
// contains just a pointer to this data and a length
// No heap allocation — zero GC cost
```

### Comparing with Interpreted Strings in the Binary

```go
const a = `\n`    // .rodata: 5c 6e (backslash, n) — 2 bytes
const b = "\n"   // .rodata: 0a (newline) — 1 byte
```

Both are compile-time constants. The difference is only in which bytes are stored.

## 4. Performance Architecture

### Large Raw String Constants

```go
// Embedding large data as raw string constants is a common pattern
// for embedding files before go:embed existed
const htmlTemplate = `
<!DOCTYPE html>
<html>
... (thousands of lines) ...
</html>
`
```

For production use, prefer `//go:embed` over large raw string constants:
```go
import _ "embed"

//go:embed templates/page.html
var pageTemplate string
// This is loaded from a file at compile time — same .rodata storage,
// but separates template files from Go source
```

### Raw Strings in Regex: Compilation Overhead

```go
// BAD: re-compiles on every call
func validate(s string) bool {
    re := regexp.MustCompile(`^\d{3}-\d{4}$`)
    return re.MatchString(s)
}

// GOOD: compile once at package initialization
var phoneRe = regexp.MustCompile(`^\d{3}-\d{4}$`)
func validate(s string) bool {
    return phoneRe.MatchString(s)
}

// The raw string itself has zero cost — it's the regexp.Compile that's expensive
// (regexp.Compile: ~10µs, regexp.Match: ~200ns)
```

## 5. Architecture Patterns

### Pattern 1: Query Repository Pattern

```go
// db/queries/queries.go
package queries

// All SQL queries as package-level constants
// Raw strings allow copy-paste between Go and psql/pgAdmin
const (
    UserByID = `
        SELECT id, name, email, created_at, updated_at
        FROM users
        WHERE id = $1 AND deleted_at IS NULL
    `

    UsersByTeam = `
        SELECT u.id, u.name, u.email
        FROM users u
        JOIN team_members tm ON tm.user_id = u.id
        WHERE tm.team_id = $1
          AND u.deleted_at IS NULL
          AND tm.deleted_at IS NULL
        ORDER BY u.name
    `

    CreateUser = `
        INSERT INTO users (id, name, email, password_hash, created_at)
        VALUES ($1, $2, $3, $4, NOW())
        RETURNING id, created_at
    `
)

// db/repository.go
type UserRepository struct {
    db *sql.DB
}

func (r *UserRepository) FindByID(ctx context.Context, id int64) (*User, error) {
    row := r.db.QueryRowContext(ctx, queries.UserByID, id)
    // ...
    return nil, nil
}
```

**Benefits**:
- Queries can be tested in database tools without modification
- Easy to review SQL separately from Go logic
- Raw strings preserve SQL formatting for readability

### Pattern 2: Embedded Template Registry

```go
// templates/registry.go
package templates

import (
    "strings"
    "text/template"
)

var Registry = map[string]*template.Template{}

func init() {
    for name, tmpl := range rawTemplates {
        t := template.Must(template.New(name).Parse(tmpl))
        Registry[name] = t
    }
}

// rawTemplates holds all templates as raw string literals
var rawTemplates = map[string]string{
    "welcome_email": `
Dear {{.Name}},

Welcome to {{.AppName}}! Your account has been created.

Login at: {{.LoginURL}}

Best,
The {{.AppName}} Team
`,
    "password_reset": `
Hi {{.Name}},

Click to reset your password (expires in 24 hours):
{{.ResetURL}}

If you didn't request this, please ignore this email.
`,
}
```

## 6. Security Architecture

### Raw Strings and Injection Prevention

```go
// The structure of parameterized queries in raw strings:
// 1. Raw string defines structure (safe — no user input here)
// 2. Parameters fill values (safe via database driver)

const searchQuery = `
    SELECT id, title, content
    FROM articles
    WHERE to_tsvector('english', title || ' ' || content)
          @@ plainto_tsquery('english', $1)
      AND published_at <= NOW()
      AND deleted_at IS NULL
    ORDER BY ts_rank(
        to_tsvector('english', title || ' ' || content),
        plainto_tsquery('english', $1)
    ) DESC
    LIMIT $2 OFFSET $3
`

// The raw string makes this complex query readable
// The $1, $2, $3 placeholders are filled by the driver — safe from injection
```

### Template Injection Prevention

```go
// text/template vs html/template with raw string templates

// SAFE for HTML output: use html/template
import "html/template"

const userProfileTemplate = `
<div class="profile">
    <h1>{{.Name}}</h1>
    <p>{{.Bio}}</p>
</div>
`
// html/template auto-escapes .Name and .Bio — XSS safe

// UNSAFE for HTML output: text/template does NOT escape
import "text/template"  // DON'T use for HTML!
```

## 7. Raw Strings and Testing Architecture

### Golden File Testing

```go
// In large systems, raw strings in tests serve as "golden files"
// embedded directly in the test code

func TestGenerateConfig(t *testing.T) {
    got := GenerateNginxConfig(Config{
        Host: "example.com",
        Port: 443,
        SSL:  true,
    })

    // Golden value embedded as raw string
    want := `server {
    listen 443 ssl;
    server_name example.com;

    ssl_certificate /etc/ssl/certs/example.com.pem;
    ssl_certificate_key /etc/ssl/private/example.com.key;
}
`

    if got != want {
        t.Errorf("GenerateNginxConfig() mismatch:\ngot:\n%s\nwant:\n%s", got, want)
    }
}
```

### Table-Driven Tests with Raw String Fixtures

```go
func TestParseMultilineInput(t *testing.T) {
    tests := []struct {
        name    string
        input   string
        want    []Record
        wantErr bool
    }{
        {
            name: "valid CSV",
            input: `id,name,email
1,Alice,alice@example.com
2,Bob,bob@example.com`,
            want: []Record{
                {1, "Alice", "alice@example.com"},
                {2, "Bob", "bob@example.com"},
            },
        },
        {
            name: "empty CSV",
            input: `id,name,email`,
            want: []Record{},
        },
        {
            name: "invalid field count",
            input: `id,name,email
1,Alice`,
            wantErr: true,
        },
    }

    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            got, err := parseCSV(tt.input)
            if (err != nil) != tt.wantErr {
                t.Fatalf("parseCSV() error = %v, wantErr = %v", err, tt.wantErr)
            }
            if !reflect.DeepEqual(got, tt.want) {
                t.Errorf("parseCSV() = %v, want %v", got, tt.want)
            }
        })
    }
}
```

## 8. Postmortems & System Failures

### Incident 1: SQL Injection via String Concatenation into Raw String Template

**What happened**: A developer used a raw string as a SQL template and concatenated user input using `fmt.Sprintf`:

```go
// BUGGY code that caused the incident
func searchUsers(nameFilter string) *sql.Rows {
    query := fmt.Sprintf(`
        SELECT * FROM users
        WHERE name LIKE '%%%s%%'
    `, nameFilter)  // INJECTION VULNERABILITY!
    rows, _ := db.Query(query)
    return rows
}
```

An attacker passed `'; DROP TABLE users; --` as `nameFilter`.

**Root cause**: Using raw string as a template with `fmt.Sprintf` for user input, bypassing parameterization.

**Fix**:
```go
func searchUsers(nameFilter string) (*sql.Rows, error) {
    const query = `
        SELECT * FROM users
        WHERE name LIKE $1
    `
    return db.Query(query, "%"+nameFilter+"%") // safe: parameterized
}
```

**Lesson**: Raw strings are for the query *structure*, not for injecting values. Always use parameters for user data.

---

### Incident 2: Indentation in Raw String Caused YAML Parsing Failure

**What happened**: A Kubernetes operator embedded pod specs as raw strings. An update added indentation to the code (for readability), which added leading spaces to the YAML, breaking the parser.

```go
// Before update (worked fine):
func getPodSpec() string {
return `
apiVersion: v1
kind: Pod
metadata:
  name: worker
`
}

// After "cleanup" (YAML now has leading spaces — invalid):
func getPodSpec() string {
    return `
    apiVersion: v1
    kind: Pod
    metadata:
      name: worker
    `
}
```

**Root cause**: Developer indented the raw string for code aesthetics without realizing the leading spaces were now part of the YAML content.

**Fix**: Either use a dedent function, or keep raw strings at column 0, or use `//go:embed` with actual YAML files.

---

### Incident 3: Regex ReDoS from Assembled Pattern

**What happened**: A validation service assembled regex patterns by concatenating user-provided field names with raw string templates:

```go
// VULNERABLE:
func buildValidator(fieldName string) *regexp.Regexp {
    pattern := `^` + fieldName + `:.+$`
    return regexp.MustCompile(pattern)  // panics or hangs if fieldName is malicious
}
```

An attacker passed `(a+)+` as `fieldName`, creating a catastrophically backtracking pattern.

**Fix**: Validate that `fieldName` contains only safe characters before using it in a pattern, or use `regexp.QuoteMeta`:
```go
func buildValidator(fieldName string) *regexp.Regexp {
    // Validate fieldName first
    if !safeFieldNameRe.MatchString(fieldName) {
        return nil
    }
    pattern := `^` + regexp.QuoteMeta(fieldName) + `:.+$`
    re, err := regexp.Compile(pattern)
    if err != nil {
        return nil
    }
    return re
}

var safeFieldNameRe = regexp.MustCompile(`^[a-zA-Z][a-zA-Z0-9_]*$`)
```

## 9. Raw Strings and go:embed

Go 1.16 introduced `//go:embed` as a cleaner alternative to large raw string constants:

```go
// Old way: embed file as raw string constant (works but pollutes Go source)
const pageTemplate = `
<!DOCTYPE html>
...1000 lines...
`

// New way: embed from file (cleaner separation of concerns)
import _ "embed"

//go:embed templates/page.html
var pageTemplate string

// For multiple files:
//go:embed templates
var templateFS embed.FS

func loadTemplate(name string) string {
    data, _ := templateFS.ReadFile("templates/" + name)
    return string(data)
}
```

**When to still use raw strings**: for small, in-code templates where a separate file would be overkill (test fixtures, simple message templates, regex patterns).

**When to use go:embed**: for any template > 20 lines, for files that designers/non-Go-developers might edit, for keeping template files separately versionable.

## 10. Best Practices for Production Systems

1. **Use raw strings for all regex patterns** — prevents double-escaping bugs
2. **Use raw strings for SQL constants** — improves query readability
3. **Use `//go:embed`** for templates > ~20 lines
4. **Never `fmt.Sprintf` user input into a raw string SQL template** — always use parameterized queries
5. **Use `strings.TrimSpace` or a `Dedent` function** for raw strings with leading whitespace
6. **Keep raw string SQL constants in a `queries` package** — separates data access from logic
7. **Use `html/template` (not `text/template`) for HTML output** — even with raw string templates
8. **Compile regex patterns once** at package level, not inside functions

## 12. Testing Strategy

```go
// Test 1: Verify raw string constants don't have unexpected whitespace
func TestQueryConstants(t *testing.T) {
    // Queries shouldn't start with whitespace after TrimSpace
    queries := map[string]string{
        "UserByID":  UserByIDQuery,
        "UserList":  UserListQuery,
    }
    for name, q := range queries {
        trimmed := strings.TrimSpace(q)
        if strings.HasPrefix(trimmed, "\n") {
            t.Errorf("query %s starts with newline after TrimSpace", name)
        }
        // Verify it starts with SELECT, INSERT, UPDATE, or DELETE
        upper := strings.ToUpper(trimmed)
        validStart := strings.HasPrefix(upper, "SELECT") ||
            strings.HasPrefix(upper, "INSERT") ||
            strings.HasPrefix(upper, "UPDATE") ||
            strings.HasPrefix(upper, "DELETE")
        if !validStart {
            t.Errorf("query %s doesn't start with SQL keyword", name)
        }
    }
}
```

## 14. Summary

At the senior level, raw string literals are:
- **Binary artifacts** stored in `.rodata` with zero GC cost
- **Readability tools** for regex, SQL, and templates
- **Architectural elements** organized in query repositories and template registries
- **Security concerns** when combined with `fmt.Sprintf` and user input
- **Testing infrastructure** for golden file tests and fixtures
- **Superseded by `//go:embed`** for large embedded content in Go 1.16+
